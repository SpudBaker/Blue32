The methodology used is both lightweight and flexible (dependant on customer needs and project complexity). It can consist of the following stages potentially broken into phases (or 'Sprints');

  1. Definition of Objectives detailed in a simple document that describes success for the project in business terms.
  2. Production of 'Use-Cases', i.e 'who will be using the application and for what purpose'
  3. Data analysis, including bringing any existing data sources into the solution
  4. A 'Wire-frame' protype showing screen flows with example data
  5. Build and Test
  6. Customer Acceptance
Professionally written documentation is a given.

The set of documents can include Functional Specifications / Mock-ups, User Guides, Operational Procedures, Data Definitions.

Format can be PDF or alternatively User Guides can be created with the application as on-line help.
Simple hourly rates are available along with quoted fixed cost.

The prefered commercial arrangement is a reduced upfront cost with an ongoing usage based charge. This ensures both parties have a stake in the success of the project.
